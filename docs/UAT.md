# UAT — User Acceptance Testing (pa-clean)

Ручной чеклист для пользовательского тестирования pa-clean на финальной машине.
Цель — пройти все ключевые сценарии end-to-end, как реальный пользователь.

## 0. Prerequisites

- macOS 13+ Apple Silicon
- Python 3.11–3.13 (`uv python install 3.13`)
- Node 18+ (для пересборки WebUI)
- Mail.app и Calendar.app настроены и работают
- MLX-модель скачана локально (`mlx-community/...` или ваша)
- Automation permission будет запрошен при первом sync (одобрить)

## 1. Установка с нуля (Prod-сборка)

```bash
git clone <repo> pa-clean
cd pa-clean

# Прод-сборка БЕЗ dev-инструментов (coverage, ruff, mypy, pytest — не ставятся):
uv sync --no-dev
(cd webui && npm install && npm run build)

cp .env.example .env
$EDITOR .env    # задать PA_VAULT_PATH и PA_MLX_MODEL_PATH
```

Ожидаемый результат: `uv sync --no-dev` ставит ≈90 пакетов (runtime),
без dev-группы; `npm run build` создаёт `webui/dist/`.

- [ ] `python -c "import personal_assistant.mlx_server.server"` — без ошибок.
- [ ] `webui/dist/index.html` существует.

## 4а. Вкладка «Правила → Инструменты»: классификация (русские теги)

В Rules → Инструменты найдите редактор `classify.yaml` (textarea с заголовком
«Правила классификации» / `classify-editor`).

### Что проверять

- [ ] При первом открытии textarea **пустая** или содержит исходник из
      `data/classify.yaml`. Если файла нет — нажмите кнопку «Сохранить» один
      раз с пустым/дефолтным содержимым: бэкенд запишет `DEFAULT_CONFIG` на
      диск; перезагрузите страницу — увидите конфиг.
- [ ] В конфиге **теги на русском**:
      `urgency: срочно/важно`,
      `category: финансы/встреча/договор/требует-действия/поездка/проект/личное`.
- [ ] LLM-категории (если `llm_classify.enabled=true`) также на русском.
- [ ] Ключевые слова **двуязычные** (русские и английские): pa-clean обрабатывает
      и письма на русском, и на английском, но **выходные теги — только русские**.
- [ ] Кнопка «Применить к vault» (`classify-apply-btn`) запускает классификацию.
      Дождитесь завершения, откройте любой `.md` в `~/PersonalAssistantVault/mail/`
      и проверьте `tags:` во frontmatter: значения должны быть русскими словами,
      без английских (`urgent`, `finance` и т.п.).
- [ ] Кнопка «Сбросить теги» (`classify-reset-btn`) удаляет добавленные теги
      из vault — для повторного прогона на чистом наборе.

### Тестовый сценарий

```bash
# Создаём контрольное письмо в vault (или используем уже синкнутые):
mkdir -p ~/PersonalAssistantVault/mail/2026/05
cat > ~/PersonalAssistantVault/mail/2026/05/2026-05-26_invoice_test.md <<'MD'
---
title: "Invoice for Q2 services"
date: "2026-05-26T10:00:00+0000"
sender: "billing@vendor.com"
subject: "Срочно: оплата счёта до конца недели"
---
Please process the invoice asap. Сумма к оплате — 45 000 ₽.
MD
```

Запустить `Применить к vault` → открыть файл повторно → во frontmatter `tags:`
должны появиться `["срочно", "финансы"]` (или похожие из русского набора).

### Edit-сценарий

- [ ] Изменить в editor'е тег «срочно» на «горящее», нажать «Сохранить» →
      сообщение «classify.yaml сохранён».
- [ ] Применить к vault → проверить, что в frontmatter теперь
      `["горящее", "финансы"]`.
- [ ] Сбросить теги → исходный набор русских тегов восстановится.

## 4б. Вкладка «Правила → Инструменты»: промпты для черновика и суммаризации

Те же Rules → Инструменты, ниже classify-editor должны быть **два текстовых
поля**: «Промпт для черновика» (`draft_system`) и «Промпт для суммаризации»
(`summarize_system`). Сохраняются в `~/PersonalAssistantVault/.tool_prompts.json`.

### Что проверять — defaults

- [ ] При первом открытии textarea'и **пустые** — это значит, что используется
      встроенный дефолтный промпт. Под textarea'й (или в подсказке) должна быть
      строка «Используется дефолтный промпт» / счётчик 0 / 8000.
- [ ] Нажмите «Загрузить дефолт» (если есть кнопка) или просто перезагрузите
      страницу — поле должно остаться пустым (это значит: dataclass `draft_system`
      пуст → `effective_draft()` возвращает `DEFAULT_DRAFT_SYSTEM` из кода).
- [ ] Откройте `src/personal_assistant/services/tool_prompts.py` → проверьте, что
      `DEFAULT_DRAFT_SYSTEM` содержит развёрнутую инструкцию (анализ, структура
      ответа, чек-лист) и `DEFAULT_SUMMARIZE_SYSTEM` — структурированную
      («Решения / Задачи / Факты / Открытые вопросы»).

### Edit-сценарий — черновик

- [ ] Введите свой промпт в textarea, например: «Отвечай в одном коротком
      абзаце, без чек-листов».
- [ ] Сохранить → ToastSuccess.
- [ ] Открыть Inbox → выбрать письмо → «Создать черновик».
- [ ] Сгенерированный draft должен быть **коротким** (одно предложение / один
      абзац), без markdown-чек-листа.
- [ ] Сброс (`tp-draft-reset-btn` или похожая кнопка) → поле очищается →
      следующий черновик снова длинный со структурой.

### Edit-сценарий — суммаризация

- [ ] Введите кастомный промпт суммаризации, например: «Резюмируй в одном
      предложении на 15 слов».
- [ ] Сохранить.
- [ ] Откройте письмо или тред → «Суммаризовать».
- [ ] Ответ должен быть **одной строкой ≤15 слов** (а не структурированным с
      четырьмя заголовками).
- [ ] Сброс → следующая суммаризация снова в формате
      «Решения / Задачи / Факты / Открытые вопросы».

### Защита от prompt injection

- [ ] Попробуйте сохранить промпт с `<|system|>` или «IGNORE PREVIOUS
      INSTRUCTIONS» внутри — должна вылететь валидация
      («prompt-injection шаблон обнаружен»), сохранение отклонено.
- [ ] Промпт длиннее 8000 символов — также отклоняется.

### Делегирование (Inbox → Ассистент → 🤝 Делегировать)

Предусловие: в `Правила → Инструменты → 👥 Сотрудники для делегирования`

- [ ] Открыть письмо в Inbox → правая панель → `🤝 Делегировать` (или клавиша `D`).
- [ ] Открывается модалка с заголовком «🤝 Делегировать письмо», подсказкой
      «Кому переслать "<тема>"?» и списком коллег (radio).
- [ ] Если коллег нет в Rules — toast «Сначала добавьте сотрудников в…»
      + переход на вкладку Rules.
- [ ] Выбрать коллегу, ввести заметку («ускорь, ждут к среде»).

      с темой `Поручение: <без Re:/Fwd:>` и текстом вводной (4-7 строк).
      Внизу — флаг `🤖 Сгенерировано MLX` (или `⚙️ Шаблон без MLX`).
