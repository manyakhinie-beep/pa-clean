## 1. Первый запуск

### Установка и запуск (один раз)

```bash
# 1. Скопируйте .env-шаблон и откройте для заполнения
cp .env.example .env
$EDITOR .env                # см. §2 — обязательные поля

# 2. Полная сборка + запуск одной командой
./make.sh                   # uv sync --no-dev → webui build → pa serve

# 3. Откройте браузер
open http://127.0.0.1:8000
```

По умолчанию `make.sh` ставит **только runtime-зависимости** (без pytest/ruff/mypy) — это быстрее и работает за корпоративным прокси, который иногда блокирует свежий `coverage` или другие dev-пакеты. Полезные флаги:

| Флаг | Назначение |
|---|---|
| (без флагов) | install (--no-dev) + WebUI build + `pa serve` |
| `--no-serve` | только установить и собрать, без запуска (CI, прод) |
| `--skip-webui` | пропустить `npm run build` (взять `webui/dist/` из git) |
| `--dev` | включить группу `dev` (pytest, ruff, mypy, locust, …) |
| `--help` | справка |

После запуска вы увидите 7 вкладок навигации в левой панели. Сервер работает пока открыт терминал. Для остановки — `Ctrl+C`.

### Ежедневный запуск

После первой сборки WebUI лежит в `webui/dist/` и в git'е, так что чаще всего достаточно:

```bash
uv run pa serve            # один запуск без пересборки
# или
./make.sh --skip-webui     # запустить, не трогая npm
```

Если правили `webui/frontend/` — пересоберите:

```bash
(cd webui && npm run build)
```

## 2. Настройки

Откройте вкладку **Настройки** (иконка ⚙️ в навигации). Это первое, что нужно заполнить перед работой.

### 2.1 Профиль пользователя

Нажмите **«Профиль»** → заполните поля:

| Поле | Зачем | Пример |
|---|---|---|
| **Имя** | Отображается в интерфейсе и в приветствии Daily Brief | `Игорь` |
| **Email** | Используется для определения «мои письма» в тредах и follow-up | `igor@company.ru` |
| **Часовой пояс** | Корректное отображение дат встреч | `Europe/Moscow` |

Нажмите **«Сохранить»**. Изменения применяются немедленно.

### 2.2 MLX-модель (AI-инференс)

Поле **«Путь к модели»** — абсолютный путь к директории с загруженной MLX-моделью.

```
/Users/вы/models/mistral
```

Если модель не загружена — скачайте её (нужен HuggingFace-аккаунт):

```bash
# Mistral 7B (~4 ГБ, универсальная, хорошо работает на русском)
uv run huggingface-cli download mlx-community/Mistral-7B-Instruct-v0.3-4bit \
  --local-dir ~/models/mistral

# Или GigaChat (лучше для русского языка, ~4 ГБ)
uv run huggingface-cli download ai-sage/GigaChat-20B-A3B-instruct-4bit \
  --local-dir ~/models/gigachat
```

После сохранения пути — перезапустите сервер (`./make.sh run`). Зелёный кружок в левом нижнем углу экрана означает, что модель загружена.

> **Intel Mac:** ИИ-генерация недоступна. Vault, Inbox, Поиск и Синхронизация работают в полном объёме.

### 2.3 Vault — хранилище данных

Поле **«Путь к Vault»** — директория, куда будут сохраняться письма и встречи в формате `.md`.

```
/Users/вы/PersonalAssistantVault
```

Директория создаётся автоматически при первой синхронизации.

### 2.4 Расписание синхронизации

- **Включить автосинхронизацию** — переключатель on/off.
- **Расписание (cron)** — по умолчанию `0 9 * * *` (ежедневно в 09:00).
- Нажмите **«Сохранить расписание»**.

Расписание активируется при следующем запуске сервера.

### 2.5 Конфигурация ассистента

Вкладка **«Конфиг ассистента»** — дополнительные параметры:

| Параметр | Описание |
|---|---|
| Стиль ответов | `formal` / `casual` / `concise` |
| Язык интерфейса | `ru` / `en` |
| Максимум токенов | Длина ответа (512–4096, рекомендуется 1024) |
| Температура | Креативность (0.1 — точно, 0.9 — творчески; рекомендуется 0.3–0.5) |

## 3. Вкладка «Сегодня»

Открывается по умолчанию при запуске. Три колонки:

### Левая колонка — Daily Brief (ИИ-сводка дня)

Блок с иконкой 🤖 появляется автоматически в 08:00 (если включена автосинхронизация).

- **Кнопка ↻** — обновить сводку из кэша (мгновенно).
- **Кнопка 🤖↺** — пересоздать сводку через ИИ (занимает 5–30 секунд).
- **Кнопка 💬** — открыть Чат с вопросом «Расскажи подробнее о моём дне».

Сводка содержит: встречи дня, срочные письма, ключевые задачи и AI-инсайт.

### Центральная колонка — Встречи и события

Список событий из Apple Calendar на сегодня. Цветные точки:
- 🟢 **Зелёная** — встреча идёт прямо сейчас.
- 🔵 **Синяя** — предстоит.
- ⚫ **Серая** — уже прошла.

Кнопка **«Подготовиться»** рядом с встречей открывает Чат со сводкой: что обсуждали с участниками, открытые задачи, прошлые встречи.

Кнопка **«Открыть календарь»** (правый верхний угол) → переходит в Vault с фильтром по календарным событиям.

### Правая колонка — Требуют внимания и предложения

**«Требуют внимания»** — до 3 самых срочных писем. Клик → открывает письмо в Inbox.

**«Ассистент предлагает»** — рекомендации (например, «Ответить на письмо от Петрова», «Подготовиться к стендапу»). Клик → открывает Чат с готовым запросом.

### Создание встречи быстрым вводом

Поле **«Новая встреча...»** внизу вкладки — введите текст на русском языке:

```
Встреча с Ивановым в рабочем календаре в следующий четверг в 15:00
Созвон по проекту во вторник утром на час
Блокировать время для отчёта в пятницу 14-16
```

Появится **превью-карточка** с распознанными данными. Нажмите **✓ Создать** — событие добавится в Calendar.app.

## 4. Вкладка «Inbox»

Полноценный почтовый клиент в стиле Apple Mail × Outlook × Superhuman.

### Левая колонка — Список писем

**Фильтры** в верхней части:

| Фильтр | Что показывает |
|---|---|
| Все | Все письма и встречи |
| 🔴 Срочно | `is_urgent` (теги + правила Q1/Q3 из «Правила») |
| 🟡 Важно | `is_important` (теги + правила Q1/Q2) |
| 🔔 Ответить | `followup_needed` (правила, `action_type=EXECUTE`, теги `ответить`/`followup`) |
| Mail | Только письма |
| События | Только события Calendar |
| ↓ Дата / ↓ Приоритет | Тоггл сортировки |
| 🧵 Беседы / ☰ Список | Группировка по треду или плоский режим |
| ✓ Прочитать все | Bulk: отметить все видимые непрочитанные прочитанными |

**Полоса поиска `🔍`** — клиент-side фильтр по ФИО / email / теме /
preview / тегам. Многотокенный AND, case-insensitive, диакритика
(ё↔е). Глобальный shortcut `/` фокусирует поле, `Esc` сбрасывает.

**Группировка «Беседы» (по умолчанию):** письма с одним `thread_id`
сворачиваются под заголовком с дедуп-списком отправителей, счётчиком
`[N]` сообщений, агрегированными значками 🔴/🔔 и pill-badge'ом
непрочитанных. Шеврон ▾/▸ раскрывает тред — дочерние сообщения
отрисовываются с отступом и вертикальной направляющей слева.
Переключатель `🧵 Беседы` ↔ `☰ Список` в шапке.

**Временные секции (плоский режим):** «Сегодня / Вчера / На этой неделе /
Ранее в этом месяце / Старше» — sticky-заголовки с blur-фоном, как в
Apple Mail.

**Цветная полоска** слева от письма — приоритет (`low`/`medium`/`high`).
Срочные получают красный левый border.

**Иконка 🔔** рядом с темой — письмо ожидает вашего ответа.

**Иконка 🤖** в строке — письмо классифицировано локальным ИИ
(`ai_classified` тег).

**2-строчный preview** — фрагмент тела письма прямо в списке (как
в Apple Mail).

**Тег-пилюли** под темой — цветные метки классификатора
(`Срочно`, `Финансы`, `Проект` и т.д.). Tag-pill с классом `rule`
показывает, что флаг пришёл из ваших правил (Rules → Инструменты),
а не из frontmatter.

Клик на письмо → открывает детальный просмотр в центральной колонке.

### Центральная колонка — Детали письма

Содержит:
- Тему, отправителя, дату.
- **Тело письма** в Markdown.
- **Метаданные ИИ** (если анализ уже запущен): намерение, тон, нужен ли ответ, дедлайн, список поручений, именованные сущности.
- **Граф участников треда** (для писем с несколькими сообщениями): аватары всех участников, временная линия сообщений, индикатор «Ваша очередь ответить».
- **Вложения**: список файлов с кнопкой открытия.

**Кнопки действий** в шапке детальной панели:
- **Прочитано / Непрочитано** — переключить статус.
- **Назначить проект** — привязать письмо к проекту из вашего списка.

### Правая колонка — Ассистент

**Панель «Ассистент»** автоматически открывается при выборе письма.

Кнопки быстрых действий (горячие клавиши в скобках):

| Кнопка | Hotkey | Действие |
|---|---|---|
| **📝 Draft ответа** | `R` | Открывает Чат с полным контекстом треда и режимом «Черновик» (Stage 4) |
| **🤝 Делегировать** | `D` | Модалка с выбором коллеги (из «Правила → Инструменты»), AI генерирует короткую вводную и открывает Mail compose с заполненным To/Subject/Body |
| **📋 Сводка треда** | `U` | Отправляет в Чат запрос «Суммаризируй тред писем по теме …» с контекстом |
| **📅 Создать слот** | `C` | Открывает Чат с запросом создания события Calendar по теме письма |
| **📂 В проект** | `P` | Привязать к проекту (фокус на селектор) |
| **✓ Прочитано / ☐ Непрочитано** | — | Переключить статус |
| **📁 Архивировать** | `E` | Пометить прочитанным и убрать из списка |
| **💤 Snooze до завтра** | `S` | Отложить (тегом) |

### Делегирование: настройка и UX

1. **Один раз** — добавьте коллег в **Правила → Инструменты → 👥 Сотрудники для делегирования** (ФИО / Email / Роль / Заметка). Сохраните через «Сохранить промпты».
2. В Inbox → выбрать письмо → правая панель → **🤝 Делегировать** (или клавиша `D`).
3. В модалке выберите коллегу (radio). Опционально добавьте «Заметку для коллеги» — она встроится в вводное письмо.
4. **`👁 Предпросмотр`** — показывает тему `Поручение: …` и текст вводной (4-7 строк) без открытия Mail.
5. **`✉️ Открыть в Mail`** — открывает Mail compose-окно с заполненными `To`, `Subject`, `Body`. Вы вычитываете и нажимаете Send.

Можно изменить промпт в **Правила → Инструменты → 🤝 Промпт для делегирования** (по умолчанию формирует деловой 4-7-строчный текст на русском).

Если MLX-модель недоступна — система использует rule-based шаблон. В модалке покажется ⚙️ «Шаблон без MLX».

**Теги** — панель для ручного добавления тегов.

**Предложения ассистента** — автоматические рекомендации (например, «Ответить до конца дня», «Назначить встречу»).

## 5. Вкладка «Чат»

Диалог с локальным ИИ-ассистентом. Данные не покидают ваш компьютер.

### Режимы чата

Переключатель в верхней части центральной колонки:

| Режим | Когда использовать |
|---|---|
| **Чат** | Свободный диалог, вопросы, обсуждение |
| **Поиск** | Поиск по vault — письма, встречи, проекты |
| **Суммаризация** | Краткое изложение переписки или события |
| **Черновик** | Генерация текста письма с учётом контекста треда |

### Отправка сообщений

Введите текст в поле внизу → Enter (или Ctrl+Enter для переноса строки).

Ответ появляется токен за токеном (стриминг). Для остановки — кнопка ⏹.

### @-упоминания — прикрепить документ из Vault

В поле ввода напишите `@` → появится список файлов из vault.

```
@Отчёт_май_2026    # прикрепит письмо/документ в контекст
```

Прикреплённый документ отображается зелёным чипом над полем ввода. Ассистент будет учитывать его содержимое при генерации ответа.

### /slash-команды

Введите `/` в поле ввода для списка команд:

| Команда | Действие |
|---|---|
| `/встреча <описание>` | Открывает «Сегодня» с превью новой встречи |
| `/событие <описание>` | Аналог /встреча |
| `/суммари` | Переключает режим на «Суммаризация» |
| `/черновик` | Переключает режим на «Черновик» |
| `/поиск <запрос>` | Выполняет поиск по vault |

### Контекст треда

Когда вы открываете черновик из Inbox, в шапке чата появляется **чип 🧵** с количеством сообщений в треде.

- Нажмите **?** на чипе → откроется **Thread Drawer** с хронологией переписки и ключевыми фактами.
- Синяя полоска = входящее, зелёная = ваш ответ.
- Нажмите **×** → убрать контекст треда, вернуться к обычному чату.

### Управление тредами (левая колонка)

- **Кнопка +** — новый тред (очищает историю, начинает новый диалог).
- **Клик на тред** — вернуться к предыдущему диалогу.
- **Кнопка 🗑** рядом с тредом — удалить тред.
- **«Очистить все»** — удалить всю историю.

### Черновик → Apple Mail

После того как ассистент сгенерировал черновик письма:

1. Кнопка **«Открыть в Mail»** — создаёт compose-окно в Apple Mail с заполненным черновиком и привязкой к исходному письму (In-Reply-To).
2. Отредактируйте при необходимости.
3. Сохраните в черновики: Cmd+S (или File → Save As Draft).
4. Отправьте из Mail как обычно.

> Mail должен быть запущен. Функция работает только на macOS.

## 6. Вкладка «Vault»

Трёхколоночный просмотрщик всех синхронизированных данных.

### Левая колонка — Фильтры и дерево

**Фильтры по разделам:**
- `calendar` — встречи и события
- `mail` — письма
- `projects` — проекты
- `contacts` — контакты

**Фильтры по тегам** — кликните тег-пилюлю для фильтрации.

**Фильтры по дате** — поля «С» / «По» для временного диапазона.

**Поле поиска** — полнотекстовый поиск по заголовкам файлов.

**Кнопка ↻** (правый верхний угол) — перезагрузить vault-индекс после синхронизации.

### Центральная колонка — Список файлов

Карточки с: заголовком, датой, отправителем (для писем), тегами.

Клик → открывает содержимое в правой колонке.

Для писем с тредом — аккордеон с историей переписки (разворачивается по клику на заголовок).

### Правая колонка — Просмотр документа

Рендер `.md` файла: заголовок, метаданные, тело, список вложений.

**Секция «Упоминается в»** — показывает другие документы, которые ссылаются на этот (проекты, связанные письма).

**Кнопки действий:**
- **Открыть в Чате** — передать документ в Чат как контекст для обсуждения.
- **Суммаризировать** — открыть Чат с запросом на краткое изложение.

## 7. Вкладка «Проекты»

GTD-менеджер, связанный с vault-документами.

### Левая колонка — Список проектов

Нажмите **«+ Проект»** → введите название → Enter.

Клик на проект → открывает детали.

### Центральная колонка — Детали проекта

- **Редактировать** (карандаш) → изменить название, описание, статус.
- **«+ Задача»** → добавить подзадачу (с чекбоксом).
- **Связанные документы** → список писем/встреч из vault, привязанных к проекту.

Привязка письма к проекту: в Inbox выберите письмо → Ассистент → «Назначить проект».

### Правая колонка — Связи и AI-предложения

- **«Связанные»** — документы vault, упоминающие этот проект.
- **«Предложить цель»** — ИИ формулирует SMART-цель на основе контекста проекта.
- **«Что делать дальше»** — ИИ предлагает следующий шаг.

## 8. Вкладка «Правила»

Три суб-вкладки: **Матрица Эйзенхауэра**, **GTD-правила**, **Инструменты**.

### 8.1 Матрица Эйзенхауэра

Четыре квадранта задач:

| Квадрант | Критерий |
|---|---|
| **Срочно + Важно** | Делать немедленно |
| **Важно** | Запланировать |
| **Срочно** | Делегировать |
| **Остальное** | Устранить |

- **«+ Задача»** — добавить задачу, выбрать квадрант.
- **Клик на задачу** → перетащить в следующий квадрант.
- **«✦ AI Распределить»** — ИИ расставит задачи по квадрантам на основе тегов и дедлайнов.
- **«Сохранить»** — сохранить текущее распределение.

### 8.2 GTD-правила

**«Заметки обработки»** — свободный текст: как вы обрабатываете входящие.

**Таблица структурированных правил** — условие → действие:
```
Условие: тег = urgency:urgent  →  Действие: ответить сегодня
Условие: sender = boss@corp.ru →  Действие: прочитать первым
```

Кнопки **«Применить»** / **«Сбросить»** — применить к vault или откатить несохранённые изменения.

### 8.3 Инструменты

Три секции:

**Классификация (`classify.yaml`):**
- Редактор YAML-правил для автоматической расстановки тегов.
- **«Сохранить»** — записать изменения.
- **«Применить к vault»** — перекласcифицировать все документы немедленно.
- **«Сбросить теги»** — удалить все автоматические теги из vault.
- Примеры правил для настройки — см. файл `data/classify.yaml`.

**Системные промпты:**
- **«Промпт черновика»** — инструкция для режима «Черновик» (до 8 000 символов).
- **«Промпт суммаризации»** — инструкция для режима «Суммаризация».
- Нажмите **«Сохранить»** после изменений.

**Реестр инструментов (Tool Calling):**
- Список доступных ИИ-инструментов (например, `date_calc` — вычисление дат).
- Переключатели включения/отключения для каждого инструмента.

**ИИ-классификация (Stage 8):**
- Статистика: сколько документов классифицировано автоматически.
- **«Запустить ИИ-классификацию»** — пакетная классификация документов, которые rule-based классификатор не распознал.
- Документы с ИИ-классификацией отмечены бейджем 🤖 в Inbox.

## 10. Синхронизация данных

### Ручная синхронизация из UI

Кнопка **«Синхронизировать»** в нижнем левом углу интерфейса (иконка ↻).

Или через терминал:

```bash
./make.sh sync          # все источники (Calendar + Mail)
```

Индикатор синхронизации (нижний левый угол):
- 🟢 **Зелёный** — MLX загружен, данные актуальны.
- 🟡 **Жёлтый** — синхронизация выполняется.
- 🔴 **Красный** — MLX недоступен (чат не работает, всё остальное — работает).
- Серая надпись под индикатором — время последней синхронизации.

### Что синхронизируется

- **Apple Calendar** — **все календари** (включая read-only / подписанные: Holidays, Birthdays, iCloud-shared) за период `PA_SYNC_SOURCES` (по умолчанию 30 дней назад / 90 вперёд).
- **Apple Mail** — все письма за последние 30 дней. **Полный текст письма (body) сохраняется в vault по умолчанию** — доступен для просмотра в Inbox и для анализа ассистентом.

### Где хранятся данные

Все данные — в папке `PA_VAULT_PATH` (по умолчанию `~/PersonalAssistantVault`):

```
PersonalAssistantVault/
├── calendar/2026/05/   ← события Calendar
├── mail/2026/05/       ← письма Mail
├── contacts/           ← контакты
├── attachments/        ← вложения (если включено)
└── index.json          ← манифест для быстрого поиска
```

Оригинальные данные в Calendar.app и Mail.app **не изменяются** — только читаются.

## 11. Типовые сценарии

### Утренний обзор дня

1. Откройте **«Сегодня»** — прочитайте Daily Brief.
2. Посмотрите встречи — нажмите **«Подготовиться»** перед важной встречей.
3. Перейдите в **Inbox** → отсортируйте по приоритету → обработайте top-5.

### Ответ на срочное письмо

1. В **Inbox** найдите письмо (фильтр «Срочно» или 🔔).
2. Нажмите **«Черновик ответа»** в правой панели.
3. В Чате появится контекст всего треда → ассистент напишет черновик.
4. Нажмите **«Открыть в Mail»** → отредактируйте → Cmd+S → отправьте.

### Подготовка к встрече

1. Откройте **«Сегодня»** → найдите встречу.
2. Нажмите **«Подготовиться»**.
3. В Чате: сводка переписки с участниками, открытые задачи, история встреч.
4. Задайте уточняющие вопросы ассистенту прямо в чате.

### Создание встречи голосовым вводом

1. **«Сегодня»** → поле «Новая встреча».
2. Введите: `"Встреча с командой в пятницу в 11:00 на 2 часа"`.
3. Если календарь не указан явно (например, «в рабочем календаре»), ассистент покажет превью и попросит **выбрать календарь** из выпадающего списка перед созданием.
3. Проверьте превью → **✓ Создать**.

### Поиск документа по памяти

1. Перейдите в **«Поиск»**.
2. Введите то, что помните: `"договор поставка март Ромашка"`.
3. Откройте найденный документ → используйте кнопку **«Обсудить в Чате»**.

### Обработка незакрытых задач

1. Включите фильтр **«🔔 Ответить»** в Inbox.
2. Для каждого письма: нажмите «Черновик ответа» → отправьте.
3. После отправки через Mail — письмо пропадёт из фильтра при следующей синхронизации.

### Настройка автоклассификации

1. **Правила → Инструменты → Классификация**.
2. Откройте редактор `classify.yaml`.
3. Добавьте правило:
   ```yaml
   - name: finance
     match: ["счёт", "оплата", "платёж", "бюджет", "invoice"]
     tags: ["category:finance"]
   ```
4. Нажмите **«Сохранить»** → **«Применить к vault»**.
5. Теги обновятся во всех документах мгновенно.

## 13. Устранение проблем

### Нет ответа от ИИ (кружок крутится долго)

**Причины:**
1. Модель ещё загружается — подождите 30–60 секунд после старта сервера.
2. MLX недоступен — красный кружок в нижнем левом углу.
3. Intel Mac — MLX не работает на x86.

**Проверка:**
```bash
uv run pa check           # диагностика всех компонентов
curl http://127.0.0.1:8000/status  # статус сервера
```

### Inbox пустой после синхронизации

1. Проверьте путь к vault: `uv run pa check`.
2. Запустите синхронизацию вручную: `./make.sh sync`.
3. Перезагрузите индекс: кнопка ↻ в Vault.
4. Убедитесь, что у Mail.app есть разрешение на автоматизацию:
   **Системные настройки → Конфиденциальность → Автоматизация**.

### Vault не читает письма из Mail

1. Mail.app должен быть **запущен** во время синхронизации.
2. Разрешение: **Системные настройки → Конфиденциальность → Автоматизация** → ваш терминал → Mail.app ✓.
3. Проверьте: `uv run pa sync-mail --days-back 7`.

### Черновик в Mail открывается как новое письмо

Используйте кнопку «Черновик ответа» из **Inbox** (не из Vault или Чата напрямую). Только Inbox передаёт `message_id` для корректного `In-Reply-To`.

### WebUI выглядит неправильно (устаревший CSS)

```bash
./make.sh webui    # пересобрать CSS и JS
```

Затем обновите страницу браузера (Cmd+Shift+R — жёсткая перезагрузка).

### Поиск ничего не находит

1. Убедитесь, что vault синхронизирован и не пустой.
2. Перестройте индекс: `./make.sh build-index`.
3. Или нажмите ↻ в Vault.

### Ошибка «503 — Vault index not loaded»

Vault либо пустой, либо путь `PA_VAULT_PATH` не задан. Выполните:
```bash
./make.sh sync        # синхронизировать данные
./make.sh build-index # перестроить индекс
```

### Сервер не запускается (порт занят)

```bash
lsof -i :8000          # найти процесс
kill -9 <PID>          # завершить
./make.sh run          # перезапустить
```

Или поменяйте порт в `.env`: `PA_SERVER_PORT=8001`.
